## Задание 1

### 1.1

Если нужно восстановить данные за предыдущий день, лучше использовать полное резервное копирование. В этом случае можно полностью вернуть базу в состояние на конец прошлого дня.

### 1.2

Если нужно восстановить данные за час до поломки, лучше использовать полную резервную копию вместе с журналами изменений (WAL в PostgreSQL или бинарными логами в MySQL). Тогда можно восстановить базу на нужный момент времени.

---

## Задание 2

Пример создания резервной копии PostgreSQL:

```bash
pg_dump -U postgres mydb > backup.sql
```

Восстановление базы:

```bash
psql -U postgres mydb < backup.sql
```

---

## Задание 3

Пример инкрементного резервного копирования MySQL с помощью бинарных логов:

```bash
mysqlbinlog mysql-bin.000001 > backup.sql
```

Для инкрементного резервного копирования в MySQL обычно используют бинарные логи, в которых записываются все изменения после создания полной резервной копии.
