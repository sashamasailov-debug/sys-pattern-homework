## Задание 1

### 1.1

Для финансов и отчётов лучше использовать PostgreSQL, потому что там важна точность, надёжность и целостность данных. Также PostgreSQL поддерживает транзакции и связи между таблицами.

### 1.1*

Можно использовать CUDA для ускорения хеширования через видеокарту, так как GPU быстрее выполняет параллельные вычисления.

### 1.2

Для лендингов подойдёт MongoDB, потому что структура данных может часто меняться.
Для CRM лучше PostgreSQL, так как там много связанных данных: клиенты, сделки, заявки.

### 1.2*

Да, можно использовать PostgreSQL и хранить часть данных в JSON, чтобы совмещать гибкость и надёжность.

### 1.3

Для базы с правилами и обучением тоже подойдёт PostgreSQL, потому что структура данных простая и удобная для хранения в таблицах.

### 1.3*

Да, можно использовать ту же базу, просто сделать отдельные таблицы или отдельную схему.

### 1.4

Для логистики лучше Neo4j. Она хорошо работает со связями и маршрутами, поэтому подходит для поиска путей и связей между объектами.

### 1.4*

Закупки можно подключить туда же, если данные связаны с логистикой и поставками.

### 1.5*

Да, почти всё можно сделать на PostgreSQL, потому что она поддерживает обычные таблицы, JSON и расширения.

## Задание 2

### 2.1

1. Пользователь проходит аутентификацию.
2. Вводит номер телефона и сумму.
3. Система проверяет данные.
4. Банк проверяет деньги на карте.
5. Деньги списываются.
6. Оператор связи начисляет услугу на сумму платежа.
7. Пользователь получает сообщение об успешной операции.

Если произошла ошибка, операция отменяется.

### 2.1*

При автоплатеже всё происходит автоматически по настройкам пользователя.

1. Проверяются настройки автоплатежа.
2. Проверяется разрешение пользователя на автосписание.
3. Банк проверяет наличие денег.
4. Деньги списываются.
5. Баланс пополняется.
6. Пользователь получает уведомление.

## Задание 4

Для большого количества данных важны скорость, отказоустойчивость и масштабирование.

Подойдёт MapReduce, Hadoop или Spark, потому что они умеют распределять нагрузку между серверами и обрабатывать большие объёмы данных.
